# Датасет карт «Наместник»

Данные для цифровой версии (Godot). Источник — сканы в `Карты и ширмы/`, сверка механики с 2-й редакцией правил.

## Файлы

| Файл | Содержимое |
|---|---|
| `cards.json` | Полный пакет: schema + 64 персонажа + 24 закона |
| `characters.json` | Только персонажи |
| `laws.json` | Только законы |
| `CHARACTERS.md` | Человекочитаемая таблица персонажей |
| `LAWS.md` | Тексты законов |
| `chars_s0*.json` | Сырые выгрузки по листам (можно не использовать в игре) |

## Схема персонажа

```json
{
  "id": 32,
  "name": "КАПИТАН",
  "sectors": {"tl": "blue", "tr": "red", "bl": "red", "br": "red"},
  "levels": [
    {"level": 1, "cost": "green", "reward": {"type": "card", "amount": 1}},
    {"level": 2, "cost": "blue", "reward": {"type": "gems", "amount": 5}},
    {"level": 3, "cost": "yellow", "reward": {"type": "magic", "amount": 1}},
    {"level": 4, "cost": "yellow", "reward": {"type": "vp", "amount": 12}}
  ]
}
```

- `sectors` — цвета половинок кругов по углам карты (`tl/tr/bl/br`).
- `levels` — снизу вверх на физической карте: уровень 1 … 4.
- `cost` — цвет камня **этой** строки; при розыгрыше платятся все строки от 1 до целевого уровня (см. `ПРАВИЛА.md`).
- `reward` — награда **только** целевого уровня.

### Типы наград

| type | Значение |
|---|---|
| `vp` | Жетон очков власти `amount` |
| `gems` | `amount` камней любых цветов в резерва за ширму |
| `card` | Взять `amount` карт (законы и/или малая колода) |
| `science` / `magic` / `defense` | Жетоны на карту |
| `attack` | Жетон(ы) атаки за ширму |
| `infinite` | Неисчерпаемый камень цвета `color` на карту |
| `bonus_magic` | Бонусный жетон магии `+amount` |
| `bonus_circle` | Бонусный жетон кругов цвета `color` `+amount` |
| `multi` | Все `parts` сразу |
| `choice` | Выбрать один из `options` |

Цвета: `blue`, `red`, `green`, `yellow`.

## Схема закона

```json
{
  "id": 70,
  "sectors": {"tl": "red", "tr": "red", "bl": "yellow", "br": "green"},
  "text": "…",
  "effect": {"timing": "immediate", "summary": "…"}
}
```

В `text` иконки заменены плейсхолдерами: `[VP]`, `[VP:N]`, `[DEF]`, `[SCI]`, `[MAG]`, `[ATK]`, `[GEM:N]`, `[CARD]`, `[INF:any]`, `[BONUS_MAG:N]`, …

## Импорт в Godot

Рекомендуемый путь:

1. Скопировать `cards.json` в `res://data/`.
2. Написать `CardDatabase.gd` / `CardData` Resource, который парсит JSON при старте (или конвертировать один раз в `.tres`).
3. Не хранить игровую логику наград в UI — отдельный `RewardResolver` по `type`.

Минимальный парсер:

```gdscript
var data = JSON.parse_string(FileAccess.get_file_as_string("res://data/cards.json"))
var characters: Array = data["characters"]
var laws: Array = data["laws"]
```

## Достоверность

- **Законы (65–88):** тексты проверены по сканам; иконки в тексте — плейсхолдеры.
- **Персонажи (1–64):** распознаны со сканов; возможны ошибки в отдельных иконках наград.
- **КАПИТАН (#32):** стоимости и награды сверены с примером из PDF правил 2-й редакции.
- **АДМИРАЛ:** номер на скане плохо читается OCR; выставлен `id = 40` как единственный отсутствовавший номер после сверки списка 1–64. Имеет смысл перепроверить по физической карте.

Перед реализацией эффектов в коде стоит точечно сверить спорные карты со сканами в `_cards/chars/`.
